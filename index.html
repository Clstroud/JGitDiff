JGitDiff

Java utility for creating a consolidated diff report between two Git commits

Overview

This is a Java utility designed to take two commits in a (local) repository and create a more readily-digestible report of the changes that occurred between them. This is specifically designed to only report 3 types of files as of this writing:

  • .java
  • .jsp
  • .sql

This was essentially designed to meet the needs of NC State's CSC-326 (Undergraduate Software Engineering) course,but could be adapted to other types of projects if desired. The overall goal is to produce a plaintext summary of what files were changed and,if a Java class,what methods were changed.

Usage



1. Select "Open Local Repository" from the "File" menu



2. Select the top-level directory of the repository



3. Select the target base commit



4. Select the target "latest" (aka "new") commit



5. Click "Perform Diff"



The program will appear to lock (it isn't threaded) while it processes. Just give it time to do its analysis. Once finished,it will open the results in your default text editor.
